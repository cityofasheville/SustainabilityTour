Installing an Configuring the Map Tour Application

This read-me file explains how to set up the Map Tour application to run on your web server and modify its content to suit your purposes.

Table of Contents
  • Install the Web application
  • Configure the application

Install the Web application

These instructions assume that you have a Web server like Internet Information Services(IIS) installed and configured on your machine. If you are using another Web server, the general installation steps will be the same but you will need to check your Web server's documentation for specific information on deploying and testing the application.

  1. Copy the contents of the zip file into your web server's root directory. In IIS, the default location for the web server's root folder is c:\inetpub\wwwroot.
  2. Test the page using the following URL: http://localhost/maptour/index.html.

Configure the application

There are two basic steps in configuring the application. The first step is to format your data (locations and photos) so that the application can use it. The second step is to set the application's configuration variables so that the application will present your specific information.

  1. Formatting your Data

    The application data is stored in a CSV file, which is downloaded and read at runtime. It is therefore required that the CSV file be formatted in the expected way so the application can read it.

    2. A sample CSV file (locations.csv) has been provided. It contains 7 fields:

      Name Title to be displayed in the photo placard (bottom left of photo).
      Description Description to be displayed in the photo placard (bottom right of photo).
      Icon_type Indicates color with which photo's location will be depicted in the map. Allowable values are "R" (red) and "B" (blue). Note that numbers are assigned automatically based on order.
      Long Longitude of photo location.
      Lat Latitude of photo location.
      URL URL for photo.
      Thumb_URL URL for thumbnail of photo (used in the gallery).

    The locations.csv contains sample data which we included for your reference. Feel free to remove all the sample records and use the locations.csv as your starting data template. Once you have populated locations.csv with your own data, you're ready to move on to step 2, Setting the Configuration Variables.

  2. Setting the Configuration Variables

    Open index.html and scroll down to the section marked "begin config section". There are 8 variables that you can modify. The final variable is optional.

    3.   TITLE The title for your application. Appears at the very top of the page.
      BYLINE A one-sentence subtitle for the app. Appears toward the top of the page, below the title.
      INTRO_PICTURE First picture that the user sees. It can be a repeat of a picture in your presentation. In the intro context, however, the picture will not be tied to a location.
      INTRO_NAME Title for the intro picture.
      INTRO_DESCRIPTION Description for the intro picture.
      CSV_FILE URL for the CSV file. The CSV file will be downloaded by the application.
      BASEMAP_URL URL to the ArcGIS service that will provide the basemap.
      INITIAL_EXTENT Provides the extent at which the map will initially appear. By default, this variable is commented out. You may either specify an extent or leave the variable inactive, in which case the map will simply default to the extent of all of the locations in the CSV_FILE.

    Once you have completed the configuration section, save the file and test your application: http://localhost/maptour/index.html. It should now display your photos and locations, as well as the other information you have specified.
